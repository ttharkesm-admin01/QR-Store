# QR-Store — ระบบสแกน QR เบิกของ + ตัดสต็อกอัตโนมัติ

เว็บแอป (PWA) สำหรับห้องเก็บของขนาดเล็ก (ไม่กี่สิบรายการ) ให้พนักงาน **สแกน QR ด้วยมือถือ
→ กรอกจำนวน → ระบบตัดสต็อกให้อัตโนมัติ** พร้อมบันทึกว่า "ใครเบิกอะไร เมื่อไหร่ เท่าไหร่"
และดูประวัติย้อนหลังได้ เปิดผ่านเบราว์เซอร์มือถือได้เลย ไม่ต้องติดตั้งแอป

## ฟีเจอร์
- 📷 **สแกน QR ด้วยกล้องมือถือ** (มีช่องกรอกรหัสเองเป็นทางสำรอง)
- ➖ **ตัดสต็อกอัตโนมัติ** แบบ atomic — กันสต็อกติดลบ ปลอดภัยแม้เบิกพร้อมกัน
- 👤 **บันทึกผู้เบิก** — เข้าระบบด้วยการเลือกชื่อ + ใส่ PIN
- 📦 **จัดการของ (ผู้ดูแล)** — เพิ่ม/แก้/ลบ, เติม/ปรับสต็อก, สร้าง & พิมพ์ QR ติดชั้นวาง
- 📜 **ประวัติการเบิก/เติม** — กรองตามรายการ/คน/ช่วงวันที่ และ export CSV
- 🔔 **เตือนของใกล้หมด** บนหน้าหลัก
- 📱 ติดตั้งเป็นแอปบนหน้าจอมือถือได้ (PWA)

## เทคโนโลยี
- Next.js 16 (App Router, TypeScript) + Tailwind CSS
- PostgreSQL ผ่าน `pg` (SQL ตรง ๆ ไม่มี ORM)
- `html5-qrcode` (อ่าน QR จากกล้อง) · `qrcode` (สร้าง QR)
- เซสชันเป็น cookie ที่เซ็นด้วย HMAC, PIN เก็บแบบ hash (scrypt) — ใช้ของในตัว Node ไม่มี dependency เพิ่ม

## รันในเครื่อง (Local)
ต้องมี **Node.js 20+** และ **PostgreSQL**

```bash
# 1) ตั้งค่า environment
cp .env.example .env
#   แก้ DATABASE_URL ให้ชี้ฐานข้อมูลของคุณ และตั้ง SESSION_SECRET (สุ่มยาว ๆ)

# 2) ติดตั้ง dependency
npm install

# 3) สร้างตาราง + ใส่ข้อมูลตัวอย่าง
npm run db:setup       # = db:init (สร้างตาราง) + db:seed (ข้อมูลตัวอย่าง)

# 4) รัน
npm run dev            # เปิด http://localhost:3000
```

> หมายเหตุ: การสแกนด้วยกล้องต้องใช้ **HTTPS หรือ localhost** (ข้อกำหนดของเบราว์เซอร์)
> บนมือถือจริงให้ใช้ลิงก์ที่ deploy แล้ว (HTTPS) หรือใช้ช่องกรอกรหัสเอง

### ผู้ใช้ตัวอย่าง (จาก seed)
| ชื่อ | บทบาท | PIN |
|------|-------|-----|
| ผู้ดูแลระบบ | ADMIN | 1234 |
| สมชาย | STAFF | 1111 |
| สมหญิง | STAFF | 2222 |

## วิธีใช้งาน
1. **ผู้ดูแล**: เข้าเมนู *จัดการของ* → เพิ่มรายการ (กำหนดรหัส/ชื่อ/จำนวน/หน่วย/เกณฑ์เตือน)
2. กดปุ่ม **QR** ของแต่ละรายการ → **พิมพ์ฉลาก** ไปติดที่ชั้นวาง/กล่อง
3. **พนักงาน**: เปิดเว็บ → เลือกชื่อ + ใส่ PIN → *สแกนเบิก* → เล็ง QR → กรอกจำนวน → ยืนยัน
4. สต็อกถูกตัดทันที และดูได้ที่เมนู *ประวัติ*

## Deploy ขึ้นคลาวด์ (Vercel + Neon)
1. สร้างฐานข้อมูล Postgres ฟรีที่ [Neon](https://neon.tech) แล้วคัดลอก connection string (มี `?sslmode=require`)
2. เตรียมตารางบนฐานข้อมูลคลาวด์ (รันครั้งเดียวจากเครื่องคุณ):
   ```bash
   DATABASE_URL="<neon-url>" npm run db:init
   DATABASE_URL="<neon-url>" npm run db:seed   # (ถ้าต้องการข้อมูลตัวอย่าง)
   ```
3. นำโค้ดขึ้น GitHub แล้ว Import เข้า [Vercel](https://vercel.com)
4. ที่ Vercel → Project Settings → Environment Variables ตั้งค่า:
   - `DATABASE_URL` = connection string ของ Neon
   - `SESSION_SECRET` = ค่าสุ่มยาว ๆ (เช่น `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`)
5. กด Deploy — เสร็จแล้วเปิดลิงก์ HTTPS บนมือถือเพื่อสแกนได้เลย

## คำสั่งที่มีให้
| คำสั่ง | ทำอะไร |
|--------|--------|
| `npm run dev` | รันโหมดพัฒนา |
| `npm run build` / `npm run start` | build / รันโหมด production |
| `npm run db:init` | สร้างตารางจาก `db/schema.sql` |
| `npm run db:seed` | ใส่ข้อมูลตัวอย่าง |
| `npm run db:setup` | init + seed |

## ความปลอดภัย (ก่อนใช้งานจริง)
- เปลี่ยน `SESSION_SECRET` เป็นค่าสุ่มของคุณเอง และอย่า commit ไฟล์ `.env`
- เปลี่ยน PIN ของผู้ใช้ตัวอย่าง (หรือเพิ่มผู้ใช้จริงในตาราง `users` — เก็บ PIN เป็น hash ด้วย `hashPin`)

## โครงสร้างโปรเจกต์
```
db/schema.sql              โครงสร้างตาราง (users, items, movements)
scripts/                   init-db.mjs, seed.mjs
src/lib/                   db.ts (pool/tx), auth.ts (PIN+session), data.ts (ตรรกะสต็อก)
src/app/                   หน้าเว็บ: login, /(หน้าหลัก), scan, items, history + api/*
src/components/            Nav, QrScanner, LogoutButton
```
