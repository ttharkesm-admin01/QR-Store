# QR-Store — ระบบสแกน QR เบิกของ + ตัดสต็อกอัตโนมัติ

เว็บแอป (PWA) สำหรับห้องเก็บของขนาดเล็ก (ไม่กี่สิบรายการ) ให้พนักงาน **สแกน QR ด้วยมือถือ
→ กรอกจำนวน → ระบบตัดสต็อกให้อัตโนมัติ** พร้อมบันทึกว่า "ใครเบิกอะไร เมื่อไหร่ เท่าไหร่"
และดูประวัติย้อนหลังได้ เปิดผ่านเบราว์เซอร์มือถือได้เลย ไม่ต้องติดตั้งแอป

> ✅ **ใช้งานบนคลาวด์ล้วน ๆ ไม่ต้องรันอะไรในเครื่อง** — ตารางฐานข้อมูลถูกสร้างให้อัตโนมัติ
> และตั้งค่าผู้ดูแลคนแรกผ่านหน้าเว็บ `/setup` (เหมาะกับเครื่องบริษัทที่ติดตั้ง/รันโปรแกรมเองไม่ได้)

## ฟีเจอร์
- 📷 **สแกน QR ด้วยกล้องมือถือ** (มีช่องกรอกรหัสเองเป็นทางสำรอง)
- ➖ **ตัดสต็อกอัตโนมัติ** แบบ atomic — กันสต็อกติดลบ ปลอดภัยแม้เบิกพร้อมกัน
- 👤 **บันทึกผู้เบิก** — เข้าระบบด้วยการเลือกชื่อ + ใส่ PIN
- 📦 **จัดการของ (ผู้ดูแล)** — เพิ่ม/แก้/archive, เติม/ปรับสต็อก, สร้าง & พิมพ์ QR ติดชั้นวาง
- 👥 **จัดการผู้ใช้ (ผู้ดูแล)** — เพิ่ม/ปิดการใช้งานพนักงาน กำหนด PIN และบทบาท
- 📜 **ประวัติการเบิก/เติม** — กรองตามรายการ/คน/ช่วงวันที่ และ export CSV (กัน formula injection)
- 🔔 **เตือนของใกล้หมด** บนหน้าหลัก
- 🗂️ **เก็บประวัติครบถ้วน** — ลบรายการเป็นแบบ archive จึงไม่ทำลายประวัติการเบิกย้อนหลัง
- 📱 ติดตั้งเป็นแอปบนหน้าจอมือถือได้ (PWA)

## เทคโนโลยี
- Next.js 16 (App Router, TypeScript) + Tailwind CSS
- PostgreSQL ผ่าน `pg` (SQL ตรง ๆ ไม่มี ORM) — สร้างตารางอัตโนมัติด้วย `ensureSchema()`
- `html5-qrcode` (อ่าน QR จากกล้อง) · `qrcode` (สร้าง QR)
- เซสชันเป็น cookie ที่เซ็นด้วย HMAC, PIN เก็บแบบ hash (scrypt) — ใช้ของในตัว Node ไม่มี dependency เพิ่ม

## 🚀 Deploy ขึ้นคลาวด์ (Vercel + Neon) — ไม่ต้องใช้เครื่อง local
1. สร้างฐานข้อมูล Postgres ฟรีที่ [Neon](https://neon.tech) แล้วคัดลอก connection string (มี `?sslmode=require`)
2. นำโค้ดขึ้น GitHub แล้ว Import เข้า [Vercel](https://vercel.com)
3. ที่ Vercel → Project Settings → Environment Variables ตั้งค่า:
   - `DATABASE_URL` = connection string ของ Neon
   - `SESSION_SECRET` = ค่าสุ่มยาว ๆ (จำเป็น — ถ้าไม่ตั้ง ระบบจะไม่ยอมทำงานในโหมด production)
4. กด **Deploy**
5. เปิดลิงก์ของแอป → จะถูกพาไปหน้า **`/setup`** อัตโนมัติ → สร้างบัญชีผู้ดูแลคนแรก (ตั้ง PIN)
   - ติ๊ก "ใส่ข้อมูลตัวอย่าง" ถ้าอยากได้ของ 4 รายการไว้ทดลอง
6. เข้าเมนู **จัดการของ** เพิ่มรายการ + พิมพ์ QR, และ **ผู้ใช้** เพื่อเพิ่มพนักงาน — เสร็จแล้วให้พนักงานเปิดลิงก์เบิกได้เลย

> 📌 ตารางฐานข้อมูลถูกสร้างให้อัตโนมัติตอนเปิดใช้ครั้งแรก — ไม่ต้องรันสคริปต์/คำสั่งใด ๆ ในเครื่อง

## วิธีใช้งาน
1. **ผู้ดูแล**: *จัดการของ* → เพิ่มรายการ (รหัส/ชื่อ/จำนวน/หน่วย/เกณฑ์เตือน)
2. กดปุ่ม **QR** ของแต่ละรายการ → **พิมพ์ฉลาก** ไปติดที่ชั้นวาง/กล่อง
3. **ผู้ดูแล**: *ผู้ใช้* → เพิ่มพนักงาน (ตั้งชื่อ + PIN)
4. **พนักงาน**: เปิดเว็บ → เลือกชื่อ + ใส่ PIN → *สแกนเบิก* → เล็ง QR → กรอกจำนวน → ยืนยัน
5. สต็อกถูกตัดทันที ดูได้ที่เมนู *ประวัติ*

## รันในเครื่อง (สำหรับนักพัฒนา — ไม่จำเป็นต่อการใช้งานจริง)
ต้องมี **Node.js 20+** และเข้าถึง PostgreSQL สักตัว (เช่น Neon)
```bash
cp .env.example .env      # ใส่ DATABASE_URL และ SESSION_SECRET
npm install
npm run dev               # เปิด http://localhost:3000 แล้วไปที่ /setup
```

> หมายเหตุ: การสแกนด้วยกล้องต้องใช้ **HTTPS หรือ localhost** (ข้อกำหนดของเบราว์เซอร์)
> บนมือถือจริงให้ใช้ลิงก์ที่ deploy แล้ว (HTTPS) หรือใช้ช่องกรอกรหัสเอง

## คำสั่งที่มีให้

|--------|--------|
| `npm run dev` | รันโหมดพัฒนา |
| `npm run build` / `npm run start` | build / รันโหมด production |
| `npm run lint` | ตรวจ lint |

## ความปลอดภัย
- ต้องตั้ง `SESSION_SECRET` เสมอในโหมด production (ระบบจะ throw ถ้าไม่ตั้ง เพื่อกันการปลอม cookie)
- อย่า commit ไฟล์ `.env`
- หน้า `/setup` ใช้ได้ครั้งเดียวตอนยังไม่มีผู้ใช้ — หลังสร้างผู้ดูแลแล้วจะถูกปิดอัตโนมัติ

## โครงสร้างโปรเจกต์
```
src/lib/schema.ts          DDL ของตาราง (แหล่งความจริงเดียว ใช้สร้างตารางอัตโนมัติ)
src/lib/db.ts              pool/tx + ensureSchema()
src/lib/auth.ts            PIN (scrypt) + session (HMAC cookie)
src/lib/data.ts            ตรรกะสต็อก/ผู้ใช้ (atomic withdraw, soft-delete)
src/app/                   setup, login, /(หน้าหลัก), scan, items, users, history + api/*
src/components/            Nav, QrScanner, LogoutButton
```
